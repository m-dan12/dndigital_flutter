## 💾 Как работает сохранение

```
┌─────────────────────────────────────────────────────┐
│                  USER INTERACTION                   │
│              (Печать текста в редакторе)            │
└──────────────────┬──────────────────────────────────┘
                   ↓
┌─────────────────────────────────────────────────────┐
│            NOTE EDITOR VIEW                         │
│    (Consumer<NoteEditorViewModel>)                  │
│    onChanged() → viewModel.update*()                │
└──────────────────┬──────────────────────────────────┘
                   ↓
┌─────────────────────────────────────────────────────┐
│       NOTE EDITOR VIEWMODEL                         │
│    Дебаунс: 2 сек                                  │
│    _autoSave() → SaveNoteUseCase                    │
└──────────────────┬──────────────────────────────────┘
                   ↓
┌─────────────────────────────────────────────────────┐
│         SAVE NOTE USE CASE                          │
│    Обновляет timestamp                             │
│    → repository.saveNote(note)                      │
└──────────────────┬──────────────────────────────────┘
                   ↓
┌─────────────────────────────────────────────────────┐
│     HIVE NOTE REPOSITORY                           │
│    NoteMapper.toHiveModel(note)                    │
│    → _box.put(note.id, hiveModel)                 │
└──────────────────┬──────────────────────────────────┘
                   ↓
┌─────────────────────────────────────────────────────┐
│          HIVE DATABASE                             │
│    Бинарное сохранение на диск                     │
│    ~/.../data/[app-name]/hive/notes.hive          │
└─────────────────────────────────────────────────────┘
```

## 💡 Дизайн решения

### Почему Hive?
- ✅ Простое использование
- ✅ Быстро для локальных данных
- ✅ Встроенная сериализация
- ✅ Не требует SQL знаний
- ✅ Отлично работает на мобильных

### Почему 2-секундный дебаунс?
- ✅ Не перегружает БД при быстром печати
- ✅ Достаточно быстро для пользователя
- ✅ Экономит батарею (меньше записей)

### Почему Clean Architecture?
- ✅ Легко расширять (облако, поиск, etc)
- ✅ Легко тестировать каждый слой
- ✅ Понятная организация кода
- ✅ Масштабируемо на долгосрок
